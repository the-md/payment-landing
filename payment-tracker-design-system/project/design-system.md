# Payment Tracker — Design System (правила для Claude Code)

> **Назначение этого документа.** Это единый источник правды по дизайну Payment Tracker. Когда Claude Code создаёт новые экраны, компоненты или фичи — он должен сначала прочитать этот файл и следовать токенам и паттернам ниже. Токены и паттерны обязательны. Layout и композиция — на усмотрение, в рамках правил.
>
> **Как читать этот документ.** Разделы идут от низкоуровневых (цвет, типографика) к высокоуровневым (экраны, копирайтинг). В конце — чек-лист для нового экрана и список анти-паттернов.

---

## 0. TL;DR для LLM

Если нет времени читать всё — вот обязательный минимум:

1. **Стек**: Next.js 14 App Router + Tailwind 3 + shadcn/ui + Recharts + date-fns (`ru` locale).
2. **Цвета — только Tailwind-токены.** Primary: `blue-600`. Canvas: `slate-50`. Карточки: белые. Текст: `slate-900 / slate-600 / slate-400`. Границы: `slate-200`.
3. **Семантическая тройка**: просрочено = `red`, ожидается = `yellow`, оплачено = `green`. Всегда вместе. Нигде не используй эти цвета для не-статусной роли.
4. **Шрифт один**: Inter. Кириллица + латиница. Никаких других семейств.
5. **Иконки только Lucide**, outline, `stroke-width: 2`. Никаких эмодзи, никаких декоративных юникод-символов.
6. **Скругления**: `rounded-lg` (кнопки, инпуты) → `rounded-xl` (карточки, модалки) → `rounded-full` (аватары, точки-статусы).
7. **Тени минимальны**: карточки = без тени, только border. Модалки = `shadow-xl`. Всё.
8. **Никаких градиентов, блюров, текстур, анимированных фонов.**
9. **Язык интерфейса — русский.** Даты, валюта (₽), числа (`ru-RU`, неразрывные пробелы как разделители тысяч).
10. **Обращение — «вы» с маленькой буквы.** Деловой, но тёплый тон.

Если генерируемый код это нарушает — перепиши, прежде чем выдавать пользователю.

## 1. Логотип и брендинг

### 1.1 Основная марка

Марка — три вертикальных столбика + акцентная точка. Источник правды: `public/logo.svg` (копия оригинала: `logo/logo.svg` в этом проекте).

**Геометрия (viewBox 64×64) — зафиксирована, не менять:**

| Элемент | Координаты | Цвет |
|---|---|---|
| Столбик 1 (низкий, светлый) | `x=8 y=26 w=14 h=20 rx=4` | `#60a5fa` |
| Столбик 2 (высокий, основной) | `x=25 y=14 w=14 h=32 rx=4` | `#2563eb` |
| Столбик 3 (средний, тёмный) | `x=42 y=20 w=14 h=26 rx=4` | `#1e3a8a` |
| Акцентная точка | `cx=49 cy=26 r=3.5` | `#ff6b4a` |

### 1.3 Где и в каком размере использовать

| Контекст | Размер | Примечание |
|---|---|---|
| Шапка приложения (авторизованная зона) | 28–32 px | рядом с wordmark «Payment Tracker», Inter 600, `text-slate-900` |
| Шапка лендинга | 32 px | в скруглённом контейнере 40×40 `rounded-lg` если нужен фон |
| Favicon | 16, 32 px | `app/icon.svg` в Next.js app router |
| Touch-icon iOS | 180×180 | иконка в скруглённом контейнере с фоном `#fff`, охранное поле ≥ 15% |
| OG/социальные превью | в контейнере 512×512+ | иконка занимает 50–60% стороны |
| Пустой аватар / заглушка | не использовать | марка — это продукт, а не персона |

### 1.4 Охранное поле и минимальный размер

- **Охранное поле:** со всех сторон минимум высоты среднего столбика (≈ 32 units в системе viewBox).
- **Минимальный размер:** 16 px. Меньше — форма разваливается, акцентная точка пропадает.

### 1.5 Разрешённые трансформации

Ничего сверх перечисленного делать нельзя.

- ✅ Вращать нельзя, наклонять нельзя
- ✅ Можно поместить в контейнер с фоном (белый, slate-50, ink) — иконка не меняется
- ✅ Монохромный вариант: все 4 элемента красятся в `currentColor` (один цвет), геометрия сохраняется — для печати и ч/б контекстов

### 1.6 Запрещено

- Менять порядок, высоту или цвет столбиков
- Убирать или перекрашивать коралловую точку
- Добавлять тень, обводку, градиент на элементы марки
- Анимировать в шапке (марка статична; разрешена лёгкая анимация только как easter egg на 404-странице или в лоадере)
- Помещать на цветной фон, не входящий в палитру дизайн-системы
- Растягивать непропорционально, обрезать viewBox
- Использовать как декоративный паттерн (повтор, плитка)

### 1.7 Wordmark

Рядом с маркой пишем «**Payment Tracker**» шрифтом **Inter 600**, цвет `text-slate-900` (или `text-white` на тёмном). Отступ между маркой и текстом — `gap-2` (8 px) при размере марки 28–32 px.

Сокращения «PT», «PayTrack» и т. п. не использовать — имени в марке нет, и в тексте нужно имя полностью.

## 2. Цвета

### 1.1 Палитра

Все цвета из Tailwind по умолчанию. Не добавляем кастомные hex'ы — только если дизайнер явно утвердил.

**Primary (акцент, CTA, активные состояния, ссылки, фокус)**
- `blue-50`  `#eff6ff` — фон подсвеченной nav-item, стат-карточка «клиенты»
- `blue-100` `#dbeafe` — аватар-плашка (фон)
- `blue-500` `#3b82f6` — focus ring, точки в графиках
- `blue-600` `#2563eb` — **основной цвет**: кнопки, активный текст nav-item, выбранный день в календаре
- `blue-700` `#1d4ed8` — hover у primary-кнопок, текст на blue-100 фоне

**Нейтралы (slate, 50→900)** — весь каркас UI: фон, текст, границы.
- `slate-50`  — фон страницы (`bg-slate-50`)
- `slate-100` — разделители внутри карточек (`divide-y`)
- `slate-200` — границы карточек/инпутов (`border-slate-200`)
- `slate-300` — disabled границы, placeholder стрелочки
- `slate-400` — вспомогательный текст, плейсхолдеры (`placeholder:text-slate-400`)
- `slate-500` — второстепенный текст, подписи, метаданные
- `slate-600` — основной нейтральный текст (body)
- `slate-700` — заголовки внутри карточек
- `slate-900` — главные заголовки, значения в стат-карточках

**Семантическая тройка статусов платежей и состояний**. Использовать **строго** только для статусов/алертов, не декоративно.

| Роль | Light BG | Border | Text | Dot | Применение |
|---|---|---|---|---|---|
| Просрочено (`overdue` / `error`) | `red-50` | `red-100/200` | `red-600/700` | `red-500` | просроченные платежи, deadline пропущен, destructive-кнопки |
| Ожидается (`pending` / `warning`) | `yellow-50` | `yellow-100/200` | `yellow-700` | `yellow-400` | ожидаемые платежи, «через N дней» |
| Оплачено (`paid` / `success`) | `green-50` | `green-100/200` | `green-700` | `green-500` | полученные платежи, подтверждения |

**Зарезервированные цвета (НЕ использовать без особой причины)**: purple, pink, orange, teal, indigo. Если появляется новая роль — обсудить с дизайнером, не изобретать.

### 1.2 CSS-переменные

Реальные значения живут в `colors_and_type.css` как CSS custom properties. В коде приложения — обращаемся через Tailwind-классы. CSS-переменные нужны только если инлайн-SVG или canvas требует raw hex.

### 1.3 Dark mode

Dark mode обязательно поддерживается. Используем стратегию `class="dark"` на `<html>`. Токены:

| Role | Light | Dark |
|---|---|---|
| Canvas (страница) | `slate-50` | `slate-950` (`#020617`) |
| Карточка | `white` | `slate-900` (`#0f172a`) |
| Граница | `slate-200` | `slate-800` |
| Разделитель | `slate-100` | `slate-800` |
| Текст основной | `slate-900` | `slate-50` |
| Текст body | `slate-600` | `slate-300` |
| Текст вспом. | `slate-500` | `slate-400` |
| Текст placeholder | `slate-400` | `slate-500` |
| Hover-фон (nav, row) | `slate-50` | `slate-800` |
| Primary solid | `blue-600` | `blue-500` (чуть светлее для контраста) |
| Primary soft bg | `blue-50` | `blue-950/40` |
| Primary soft text | `blue-700` | `blue-300` |
| Success soft bg | `green-50` | `green-950/40` |
| Success soft text | `green-700` | `green-300` |
| Warning soft bg | `yellow-50` | `yellow-950/40` |
| Warning soft text | `yellow-700` | `yellow-300` |
| Danger soft bg | `red-50` | `red-950/40` |
| Danger soft text | `red-600` | `red-300` |

**Правила dark mode:**
- Всегда парная запись: `bg-white dark:bg-slate-900`, `text-slate-900 dark:text-slate-50` и т.д.
- **Не** использовать `dark:bg-black` — всегда `slate-950` минимум (иначе карточки сливаются).
- Границы в dark: `dark:border-slate-800`, не `dark:border-slate-700` (слишком ярко).
- Focus ring в dark — оставляем `blue-500`, работает на обоих темах.
- Тени в dark практически не видны — **не полагаться на тень** как на разделитель; использовать border.
- Shadcn-компоненты, собранные через CLI, уже поддерживают dark — не переопределять.

**Переключатель темы** живёт в настройках пользователя. Дефолт — системная (`prefers-color-scheme`).

## 2. Типографика

### 2.1 Семейство

**Inter.** Единственное семейство. Подсеты: `latin` + `cyrillic`. Веса: 400, 500, 600, 700.

Fallback: `-apple-system, BlinkMacSystemFont, system-ui, sans-serif`.

```css
font-family: 'Inter', -apple-system, BlinkMacSystemFont, system-ui, sans-serif;
```

Для чисел (суммы, статистика, таблицы) — `font-variant-numeric: tabular-nums` (в Tailwind: `tabular-nums`). Обязательно для всех денежных значений и колонок с числами.

### 2.2 Шкала

| Роль | Класс | Размер | Вес | Использование |
|---|---|---|---|---|
| Display | `text-3xl font-bold` | 30px | 700 | редко, лендинги |
| H1 / Page title | `text-2xl font-bold` | 24px | 700 | заголовок страницы |
| H2 / Section | `text-xl font-semibold` | 20px | 600 | секции внутри страницы |
| H3 / Card title | `text-base font-semibold` | 16px | 600 | заголовок карточки, модалки |
| Body | `text-sm` | 14px | 400 | основной текст, строки таблиц |
| Label | `text-sm font-semibold` | 14px | 600 | подписи полей формы |
| Caption / Meta | `text-xs` | 12px | 400 | второстепенная инфа, подписи графиков |
| Micro | `text-[11px]` | 11px | 400 | бейджи, мельчайшие подписи. Меньше — нельзя. |

**Межстрочные расстояния** оставляем Tailwind-дефолтные. Кастомить `leading-*` только в исключительных случаях.

**Цвета текста по иерархии** (повтор из §1 для удобства):
- Заголовки H1–H3 → `text-slate-900`
- Body → `text-slate-600` (или `slate-700` для более тёмного)
- Meta / caption → `text-slate-500` или `text-slate-400`

## 3. Spacing, радиусы, тени

### 3.1 Spacing

Шкала Tailwind (4px base). Используем эти значения, других — не добавляем:

`1` (4) · `2` (8) · `3` (12) · `4` (16) · `5` (20) · `6` (24) · `8` (32) · `10` (40) · `12` (48)

**Правила:**
- Gap между элементами формы → `space-y-4` (16px) внутри колонки.
- Padding карточки → `p-5` или `p-6` (20/24px). Никогда меньше `p-4`.
- Padding страницы → `p-6 md:p-8` (24/32px).
- Gap в строке с иконкой → `gap-2` (8px) для `text-sm`, `gap-3` (12px) для `text-base`.
- Отступ между секциями на странице → `space-y-6` (24px).

### 3.2 Радиусы

| Класс | Размер | Где |
|---|---|---|
| `rounded` | 4px | бейджи-тэги (`tag-overdue`, `tag-soon`) |
| `rounded-md` | 6px | внутренние мелкие элементы |
| `rounded-lg` | 8px | **кнопки, инпуты, селекты, nav-item** |
| `rounded-xl` | 12px | **карточки**, стат-карточки, toast |
| `rounded-2xl` | 16px | **диалоги/модалки**, empty-state контейнер, auth-card |
| `rounded-full` | 999px | аватары, точки-статусы, пилюли-счётчики в табах |

Не смешивать. Карточка всегда 12px, кнопка внутри — 8px.

### 3.3 Тени

Используем по минимуму. Иерархия глубины создаётся **бордерами**, не тенями.

- Карточки на странице → **без тени**, `border border-slate-200`.
- Dropdown / Popover → `shadow-md`.
- Модалки → `shadow-xl`.
- Toast → `shadow-lg`.
- Кнопки — без тени (только у `.google-btn` в login — `shadow-sm`).

## 4. Компоненты — правила

### 4.1 Кнопки

Варианты: `primary` | `ghost` | `danger`. Размеры: `md` (по умолчанию) | `sm`.

- **Primary**: `bg-blue-600 text-white hover:bg-blue-700` — CTA, главное действие на экране/модалке. **Одна primary-кнопка на экран/модалку.**
- **Ghost**: `bg-white border border-slate-200 text-slate-600 hover:bg-slate-50` — второстепенные действия, «Отмена».
- **Danger**: `bg-red-600 text-white hover:bg-red-700` — только деструктивные действия (удалить, отменить подписку).
- Общее: `rounded-lg font-semibold transition-colors`, `min-h-[40px]` (md) / `min-h-[30px]` (sm).
- Loading state → `disabled` + текст заменяется на «...» или spinner.
- Иконка в кнопке: `gap-2`, иконка слева, `w-4 h-4`.

### 4.2 Формы (инпуты, селекты, textarea)

- `rounded-lg border border-slate-200 px-4 py-2.5 text-sm`.
- Focus: `focus:border-blue-500 focus:ring-2 focus:ring-blue-500/20`.
- Placeholder: `placeholder:text-slate-400`.
- **Label сверху**, `text-sm font-semibold text-slate-700 mb-1.5`. Никаких floating labels.
- **Helper-текст** — снизу, `text-xs text-slate-500 mt-1`.
- **Ошибка валидации** — рамка `border-red-500`, сообщение снизу `text-xs text-red-600`.
- **Обязательное поле** — звёздочка после лейбла: `Название *`. Не менять цвет лейбла.
- Ширина — full-width внутри колонки формы. Короткие поля (сумма, дата) — `grid grid-cols-2 gap-4`.

### 4.3 Карточки

- `bg-white border border-slate-200 rounded-xl`.
- Padding: `p-5` или `p-6`.
- Разделитель между строками внутри карточки: `divide-y divide-slate-100`.
- **Не** использовать `shadow-*` на обычных карточках.
- Dark: `dark:bg-slate-900 dark:border-slate-800`.

### 4.4 Бейджи и статусы

- Пилюля: `inline-flex items-center gap-1 rounded-full px-2 py-0.5 text-[11px] font-semibold border`.
- Статус клиента: `active` (зелёный), `paused` (жёлтый), `completed` (серый).
- Статус платежа: `paid`, `pending`, `overdue` — цвета из семантической тройки.
- **Точка-статус** (`w-2 h-2 rounded-full`) — используется в списках и календаре, когда нет места для пилюли.
- Тэг «5 дн. просрочено» / «через 3 дн.» — `rounded` (4px), не `rounded-full`, чтобы отличать от статус-пилюль.

### 4.5 Таблицы и списки

В этом проекте **нет классических таблиц** — используем список карточек с flex-строками. Правила:
- Разделители между строками → `divide-y divide-slate-100`.
- Hover на строке → `hover:bg-slate-50`, `cursor-pointer` если кликабельна.
- Аватар слева, контент в середине с `flex-1 min-w-0`, числа/статус справа.
- Сумма и метаданные справа — `text-right`.
- Шеврон `>` в конце строки → `text-slate-400` (только если строка ведёт на детальную).

**Пустое состояние списка** — см. §4.9.

### 4.6 Модалки / Диалоги

Используем shadcn `Dialog`. Всегда `rounded-2xl` + `shadow-xl` + overlay `bg-black/40`.

Три размера — выбор по контенту:

| Размер | Ширина | Когда |
|---|---|---|
| **Small** | `max-w-sm` (384px) | Confirm-диалоги («Удалить?», «Отметить оплату?»). Только заголовок + 1–2 предложения + 2 кнопки. |
| **Medium** | `max-w-md` (448px) | Простые формы в 1 колонку (добавить клиента, изменить пароль). |
| **Large** | `max-w-2xl` (672px) | Сложные формы в 2 колонки, детальный просмотр клиента, редактирование платежа с историей. |

**Никогда не использовать** `max-w-full` или больше `2xl` — если контент не помещается, это отдельная страница, не модалка.

**Структура модалки:**
- Header: заголовок (`text-base font-semibold`) + крестик-закрытие справа. Крестик — всегда есть, кроме destructive-confirm.
- Body: `p-6`, `space-y-4`.
- Footer: две кнопки, `flex gap-3`, primary справа (в русском UI — чтение слева-направо, главное действие справа).

**Destructive-confirm** — primary-кнопка становится `danger` (красная). Текст: «Удалить», «Отменить подписку». Ghost-кнопка слева: «Отмена».

**Модалка vs страница:**
- Быстрое действие, не требует контекста → модалка.
- Длинная форма (>8 полей) → отдельная страница или drawer.
- Редактирование с preview → страница.

### 4.7 Навигация

**Desktop sidebar** — фиксированный слева, 256px, `bg-white border-r border-slate-200`.
- Nav-item: `flex items-center gap-3 rounded-lg px-3 py-2.5 text-sm font-medium`.
- Active: `bg-blue-50 text-blue-600`.
- Hover (неактивный): `hover:bg-slate-50 hover:text-slate-900`.
- Иконка слева, 20×20, Lucide.
- Внизу sidebar'а — user-row с аватаром, именем, email и кнопкой logout.

**Mobile nav** — bottom tab bar, `fixed bottom-0`, высота `h-16`, `bg-white border-t border-slate-200`. 4–5 пунктов. Иконка 24×24 сверху, подпись `text-[11px]` снизу. Active — `text-blue-600`.

### 4.8 Табы (внутри страницы)

Нижнее подчёркивание, не пилюли.
- Контейнер: `flex border-b border-slate-200`.
- Таб: `px-4 py-2.5 text-sm font-semibold border-b-2 -mb-px`.
- Active: `border-blue-600 text-blue-600`.
- Inactive: `border-transparent text-slate-500 hover:text-slate-700`.
- Счётчик (pill) справа: `rounded-full bg-slate-100 px-2 py-0.5 text-[11px]`. Active — `bg-blue-100 text-blue-700`.

### 4.9 Пустые состояния (empty states)

- Dashed-border контейнер: `border-2 border-dashed border-slate-200 rounded-2xl bg-white p-12 text-center`.
- Иконка в круге: `w-16 h-16 rounded-full bg-blue-50 flex items-center justify-center mx-auto mb-4` + Lucide-иконка `w-8 h-8 text-blue-500`.
- Заголовок: `text-lg font-semibold text-slate-700`.
- Описание: `text-sm text-slate-500 max-w-xs mx-auto`.
- CTA: primary-кнопка.

Используем **везде**, где список пустой (не только на дашборде): «Нет клиентов», «Нет платежей в этом месяце», «Поиск ничего не нашёл».

### 4.10 Toast-уведомления

Используем shadcn `sonner` (или `Toaster`), **позиционирование — bottom-right на desktop, bottom-center на mobile**.

- Размер: `rounded-lg px-5 py-3 text-sm shadow-lg`, максимум 2 строки.
- Варианты:
  - `success` — `bg-green-600 text-white`, иконка `Check`. Длительность: **2500 мс**.
  - `info` — `bg-slate-900 text-white`, без иконки. **2500 мс**.
  - `warning` — `bg-yellow-500 text-slate-900`, иконка `AlertTriangle`. **4000 мс**.
  - `error` — `bg-red-600 text-white`, иконка `AlertCircle`. **5000 мс** (+ кнопка «Повторить», если применимо).
- **Одновременно видно максимум 3 тоста.** Новые вытесняют старые снизу.
- **Когда что показывать:**
  - Успешное действие (сохранили, отметили, удалили) → `success`, коротко: «Клиент добавлен», «Платёж отмечен».
  - Асинхронная ошибка (сеть, сервер) → `error` с кнопкой «Повторить».
  - Предупреждение об ограничении тарифа → `warning`.
  - Информационные события (копировалось в буфер, ссылка скопирована) → `info`.
- **Не использовать тост** для: валидации форм (показываем inline), подтверждения деструктивных действий (используем модалку-confirm), длинных сообщений (>2 строк — это alert-карточка или модалка).
- Dark: toast цвета инвертируем: `success` → `bg-green-500`, `info` → `bg-slate-100 text-slate-900`.

### 4.11 Loading & Skeleton

Три стратегии по длительности ожидания:

| Ожидание | Стратегия |
|---|---|
| < 300 мс | Ничего не показываем (иначе мерцает). |
| 300 мс – 2 с | **Skeleton-плейсхолдер** в форме контента. |
| > 2 с | Skeleton + пояснительный текст («Загружаем платежи за февраль...»). |
| Блокирующая операция (сохранение формы) | Кнопка → `disabled` + `...` вместо текста. **Не блокируем** весь UI оверлеем. |

**Skeleton-стили:**
- `bg-slate-200 rounded animate-pulse` (свет) / `dark:bg-slate-800`.
- **Форма скелетона повторяет форму реального контента** — аватар (круг), строки текста (полосы), кнопка (прямоугольник). Не одна серая плашка на всё.
- Высота полос текста: `h-4` для body, `h-5` для заголовков. Ширина — варьировать (60%, 80%, 90%), чтобы не выглядело как табуляция.
- Не использовать shimmer-анимацию — только `animate-pulse` (shadcn-дефолт).

**Spinner** — только внутри кнопок и очень коротких инлайн-индикаторов. Для больших областей — skeleton.

```
// Пример: список платежей
<div className="space-y-2">
  {Array.from({length: 5}).map((_, i) => (
    <div key={i} className="flex items-center gap-4 p-4 border rounded-xl">
      <div className="w-10 h-10 rounded-full bg-slate-200 animate-pulse" />
      <div className="flex-1 space-y-2">
        <div className="h-4 w-1/3 bg-slate-200 rounded animate-pulse" />
        <div className="h-3 w-1/2 bg-slate-200 rounded animate-pulse" />
      </div>
      <div className="h-5 w-20 bg-slate-200 rounded animate-pulse" />
    </div>
  ))}
</div>
```

### 4.12 Графики (Recharts)

- Столбчатые → `fill="#2563eb"`, `radius={[4,4,0,0]}`.
- Линейные → `stroke="#2563eb"`, `strokeWidth={2}`, `dot={{fill:'#2563eb', r:3}}`.
- Сетка → `stroke="#f1f5f9"`, `strokeDasharray="3 3"`.
- Оси → `tick={{fill:'#94a3b8', fontSize:11}}`, без линии (`axisLine={false}`).
- Tooltip — белая карточка `rounded-lg border border-slate-200 shadow-md p-3 text-xs`.
- **Если серий несколько**: blue-600 → blue-400 → slate-400. Не вводить новые цвета без необходимости.

## 5. Адаптивность

### 5.1 Брейкпоинты (Tailwind-дефолты)

| Имя | Ширина | Устройство |
|---|---|---|
| `sm` | 640px | большой мобильный (альбомная) |
| `md` | 768px | планшет (портрет) |
| `lg` | 1024px | планшет (альбом) / маленький ноут |
| `xl` | 1280px | desktop |
| `2xl` | 1536px | большой desktop |

**Mobile-first.** Базовые стили — мобильные, расширяем через `md:`, `lg:`.

### 5.2 Ключевые переходы

- **Sidebar → bottom tab bar**: на `<md` скрываем `aside`, показываем `nav` снизу. `hidden md:flex` / `md:hidden flex`.
- **Page padding**: `p-4 md:p-6 lg:p-8`.
- **Стат-карточки grid**: `grid-cols-1 md:grid-cols-3 gap-4`.
- **Форма в модалке**: `grid grid-cols-1 md:grid-cols-2 gap-4` — поля в 2 колонки только от `md`.
- **Таблица-подобные списки** на мобильном: скрываем второстепенные колонки (контактный телефон), оставляем имя + сумма + статус.

### 5.3 Мобильные правила

- **Hit target** — минимум 44×44px. Кнопки `sm` не использовать на touch-устройствах (используем `md`).
- **Fixed bottom-nav** забирает 64px снизу — добавляем `pb-16 md:pb-0` на main-container.
- **Модалки на мобильном** — full-screen или bottom-sheet (`max-w-full h-full rounded-none` / `rounded-t-2xl` снизу). Для `small`-confirm — всё равно центральный диалог.
- **Тосты** — `bottom-center`, ширина `max-w-[calc(100vw-32px)]`.
- **Таб-бары внутри страницы** — скроллятся горизонтально (`overflow-x-auto`) если >3 табов.

## 6. Иконки

**Только Lucide** (`lucide-react`). Outline, `stroke-width: 2`, `strokeLinecap: round`, `strokeLinejoin: round`.

**Размеры:**
- `w-4 h-4` (16px) — внутри текста `text-sm`, внутри кнопок.
- `w-5 h-5` (20px) — sidebar nav, префикс в инпуте.
- `w-6 h-6` (24px) — mobile bottom nav, стандалон в тулбаре.
- `w-8 h-8` (32px) — крупная иконка в empty-state (внутри `w-16 h-16` круга).

**Цвет:** `text-slate-500` по умолчанию (inherit от родителя). В кнопках наследует `currentColor`. Никогда не задаём цвет hex-ом.

**Запрещено:**
- Эмодзи в UI (`😀`, `✅`, `💰`). Только в пользовательских текстах (имена клиентов, комментарии), никогда в системных надписях.
- Юникод-символы-как-иконки: `→`, `✓`, `★`. Исключение — разделитель-точка `·` между метаданными.
- Рисовать SVG-иконки вручную. Если нужной иконки нет в Lucide — обсудить добавление свежей или найти альтернативу.

## 7. Копирайтинг — тон и голос

**Аудитория.** Владельцы и менеджеры маркетинговых агентств + малый бизнес. Прагматики, ценят экономию времени. Русскоязычные. Не любят канцелярит и не любят чрезмерную фамильярность.

**Тон: деловой, но тёплый.**
- Обращение: **«вы» с маленькой буквы** (не «Вы»).
- Короткие предложения. Без воды. Без формулировок вроде «Мы заботимся о вашем...».
- Не фамильярничаем: не «Привет!», не «🎉 Отлично!». Но и не «Уважаемый пользователь, настоящим уведомляем...».
- Глаголы в повелительном или инфинитиве: «Добавьте клиента», «Отметить оплату». Не «Вы можете добавить...».

**Казусы:**

| Хорошо ✅ | Плохо ❌ |
|---|---|
| «Добавьте первого клиента» | «Добавьте своего первого клиента, чтобы начать работу» |
| «Платёж отмечен как оплаченный» | «Вы успешно отметили платёж!» |
| «Ничего не найдено» | «К сожалению, мы не смогли найти результаты по вашему запросу» |
| «Удалить клиента?» | «Вы уверены, что хотите удалить этого клиента?» |
| «Все связанные платежи также будут удалены. Действие нельзя отменить.» | «Внимание! При удалении клиента будут удалены все его платежи. Это действие необратимо!» |
| «Через 3 дня» / «5 дн. просрочено» | «Ожидается через 3 дня» / «Просрочено на 5 дней» |

**Даты и время.**
- Короткий формат в строках: `12 фев` (d MMM, `ru` locale). Год — только если не текущий.
- Полный: `12 февраля 2026`.
- Время: `14:30` (24-часовое).
- Относительное для недавнего: «только что», «5 минут назад», «вчера», «3 дня назад». После 7 дней — переход на абсолютную дату.
- Месяцы с маленькой буквы: `февраль 2026`. В `ru` locale `date-fns` — так по умолчанию.

**Деньги.**
- Всегда с пробелом перед `₽`: `85 000 ₽`.
- Разделитель тысяч — неразрывный пробел. `Intl.NumberFormat('ru-RU')` выдаёт его автоматически.
- Без копеек в списках (`85 000 ₽`). С копейками — только на странице детали платежа (`85 000,00 ₽`).
- Отрицательные — со знаком минус, красным: `−5 000 ₽` (не `-5000`).

**Ошибки:**
- Что пошло не так + что делать. «Не удалось сохранить. Проверьте подключение и попробуйте снова.» — хорошо. «Ошибка 500» — плохо.
- Не винить пользователя: «Неверный формат» → «Укажите сумму числом, например 50000».

**Capitalization:**
- Заголовки страниц, карточек, модалок — **Sentence case**: «Добавить клиента», «Ближайшие платежи». НЕ Title Case.
- Кнопки — тоже Sentence case: «Сохранить», «Отметить оплату».
- Метки статусов — с большой буквы первое слово: «Активный», «Просрочен».

**Pluralization.**
- Русский — 3 формы. Используем `Intl.PluralRules('ru')` или `plural-ru`: `5 клиентов` / `1 клиент` / `2 клиента`.
- Не писать «1 клиент(ов)», «клиентов: 5» — только нормальные падежи.

**Чего не делаем:**
- Эмодзи в системных текстах.
- Восклицательных знаков в подтверждениях.
- CAPS LOCK.
- Слов «круто», «супер», «бомба», «вау».

## 10. Анти-паттерны (чего не делать)

1. **Не использовать purple/pink/indigo/teal** — эти цвета не в системе. Если нужен новый акцент, обсудить.
2. **Не делать градиентный primary** (`bg-gradient-to-r from-blue-500 to-purple-600`) — противоречит визуальному языку.
3. **Не ставить цветную левую границу + скругление на карточке** как декор. Цветная левая граница используется **только** в `PaymentRow` для обозначения overdue/pending.
4. **Не использовать glassmorphism**, `backdrop-blur`, полупрозрачные карточки.
5. **Не подменять статусные цвета** — `success` всегда зелёный, `error` всегда красный, не «фирменно-синий success».
6. **Не рисовать иконки вручную в SVG** — только Lucide.
7. **Не использовать эмодзи** для обозначения статусов (`✅ Оплачено`) — только Lucide `Check` + семантический цвет.
8. **Не делать сложные shadow-эффекты** — `shadow-2xl`, цветные тени, inset-тени. Максимум — `shadow-xl` на модалке.
9. **Не использовать CSS-анимации сложнее** чем `transition-colors` / `animate-pulse`. Никаких bounce, spring, кастомных keyframes.
10. **Не смешивать размеры радиусов** — карточка `rounded-xl` + кнопка внутри `rounded-lg`, а не наоборот.
11. **Не писать «Вы»** с большой буквы. Не ставить восклицательные в toast-success.
12. **Не использовать `text-center`** на длинных параграфах — только заголовки и короткие сообщения в empty-state.
13. **Не вкладывать карточки в карточки.** Если нужен внутренний блок — используй `divide-y` или `bg-slate-50`-подложку.
14. **Не использовать `shadcn` `Alert`** для всего — для временных сообщений используем toast, Alert — только для постоянных на странице (тарифные ограничения, баннеры).
15. **Не вводить новые шрифтовые размеры** вне шкалы в §2.2.
